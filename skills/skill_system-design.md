# SKILL — SYSTEM DESIGN
**Loadable by: D.S.C, D.S.E, D.I.I, D.R.A, D.S.M**

---

## WHAT THIS SKILL IS

The ability to take an intention and translate it into a functioning multi-component system with clear inputs, processes, outputs, and feedback loops. This skill is loaded whenever a new system is being conceived, designed, or redesigned.

---

## THE PROCESS

```
1. INTENTION LOCK
   State what the system is trying to accomplish in one sentence.
   This sentence governs all design decisions.

2. INPUT MAPPING
   What triggers this system? What data or material enters it?
   List every input — both obvious and hidden.

3. PROCESS ARCHITECTURE
   What happens to the inputs?
   Map the transformations step by step.
   Each step: input → action → output.

4. OUTPUT DEFINITION
   What does the system produce?
   Where does each output go?
   What quality standard does each output meet?

5. FEEDBACK LOOP
   How does the system learn from its own results?
   What metric or signal tells you the system is working?
   What metric tells you it is failing?

6. FAILURE MODE IDENTIFICATION
   What breaks first under load?
   What breaks if input quality drops?
   What happens if a component is removed?

7. BOUNDARY SETTING
   What does this system NOT do?
   Where does it end and another system begin?
```

---

## QUALITY CHECK

Before a system design is considered complete:

```
[ ] Intention is one sentence, clear, actionable
[ ] All inputs identified
[ ] Every process step has an input and output
[ ] Feedback loop exists and is measurable
[ ] At least two failure modes named and addressed
[ ] Boundaries are defined — the system has edges
```
