# WF_DII — STAGE 2: EVALUATION (ALBEDO)
**Test the technology against the sovereign standard. Five criteria. No shortcuts.**

---

## THE FIVE EVALUATION CRITERIA

**1. CAPABILITY**
What can this actually do right now? Not the roadmap — the current, demonstrated state.
Load `skill_source-evaluation.md` for suppressed or emerging technology claims.

**2. SOVEREIGNTY CHECK** *(non-negotiable)*
Does this increase or decrease Morph's independence?
- Creates vendor lock-in? → Flag. Define exit path before integrating.
- Data stored externally without control? → Flag. Requires sovereign data layer.
- Requires ongoing payment to function? → Cost must be justified by output.
- Open source with local deployment option? → Preferred.

**3. INTEGRATION FIT**
Which department(s) use this? Does it solve a problem that currently exists?
Does it replace something that works worse, or does it add complexity without value?

**4. COST / BENEFIT**
```
COST:    Time to learn + time to maintain + money + attention displaced
BENEFIT: Output quality gained + time saved + sovereignty increased
NET:     If cost > benefit after honest assessment — do not integrate.
```

**5. EVIDENCE QUALITY** *(for emerging / suppressed tech)*
What is the D.R.D source tier for its claimed capabilities?
Claims at Tier 4–5 are investigated — not integrated until evidence strengthens.

---

## OUTPUT

File as: `D.I.I/dii_eval_[technology].md`

## GATE

> Must pass sovereignty check (non-negotiable) + at least 3 of remaining 4 criteria.
> A failed sovereignty check is not a design constraint — it is a stop condition.

**PASS → load `wf_stage-3_mastery.md`**
**FAIL SOVEREIGNTY → document why. Do not proceed. Revisit if conditions change.**
