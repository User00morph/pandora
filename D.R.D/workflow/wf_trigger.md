# WF_DRD — TRIGGER & ORIENTATION
**D.R.D | Load this first. Then load only the active stage file.**

---

## WHAT THIS WORKFLOW IS

The core executable sequence of D.R.D. Takes any topic — from any department, at any scale — from raw question to fully tiered, documented research output. Runs every time research is needed rather than opinion generated.

---

## TRIGGERS

- A department file routes to D.R.D for sourcing or investigation
- A direct research request with a specific topic or question
- An existing belief or claim in the OS that needs evidence evaluation
- A new concept or framework encountered before integration

---

## INPUT REQUIRED BEFORE STARTING

```
TOPIC:             What is being researched?
ORIGIN DEPT:       Which department is requesting this?
RESEARCH DEPTH:    Surface (overview) / Standard (documented) / Deep (exhaustive)
SPECIFIC QUESTION: What specifically needs to be answered?
EXISTING POSITION: Is there already a belief held about this?
                   If yes — state it explicitly before researching.
                   Researching to confirm ≠ researching to find.
DEADLINE:          Time-sensitive or open-ended?
```

**First action every time:** Check `DRD_INDEX.md` — has this topic already been researched?
- Yes → read existing file, determine whether to extend or start parallel
- No → proceed to Stage 1

---

## THE SIX STAGES

```
STAGE 1 — DECONSTRUCT    → wf_stage-1_deconstruct.md
STAGE 2 — SOURCES        → wf_stage-2_sources.md
STAGE 3 — EVIDENCE       → wf_stage-3_evidence.md
STAGE 4 — PATTERNS       → wf_stage-4_patterns.md  (includes concept distillation)
STAGE 5 — RECONSTRUCT    → wf_stage-5_reconstruct.md
STAGE 6 — DEPLOY         → wf_stage-6_deploy.md
```

**Surface depth:** Stages 1, 4, 5 only — abbreviated. Single session.
**Standard depth:** All 6 stages. 1–3 sessions.
**Deep depth:** All 6 stages, multiple passes through 2–4. Multi-session.

Load the file for the active stage only.
