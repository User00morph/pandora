# WF_DRD — STAGE 6: DOCUMENTATION & DEPLOYMENT
**Make the research retrievable. Research not filed is research that has to be redone.**

---

## WHAT THIS STAGE IS

The reconstruction is complete. This stage packages it for the receiving department(s), files everything correctly, and updates all indexes. A research cycle that does not complete Stage 6 has not completed the workflow.

**Load: `shared/skills/skill_copywriting.md`** for formatting briefs for specific departments.

---

## ACTION

**1. CREATE THE BRIEF**
Format the reconstruction for the receiving department's needs. Each destination department gets its own brief if the framing differs.

File as: `briefs/drd_brief_[dept-code]_[topic]_deployed.md`

Brief structure:
```
TO: [Department name and code]
RE: [Topic]
CONFIDENCE: [Overall level]
KEY FINDINGS: [3–5 bullets — what this department needs to act on]
FULL FILE: [Link to reconstructed file]
OPEN QUESTIONS: [What remains unresolved — relevant to this dept]
```

**2. CROSS-LINK**
Add a reference to the brief in the destination department's context file so it is findable from both directions — from D.R.D and from the receiving department.

**3. UPDATE THE INDEX**
In `DRD_INDEX.md`:
- Move stage to `deployed`
- Move entry from ACTIVE RESEARCH to COMPLETED & DEPLOYED table
- Update DOMAIN COUNTS

**4. UPDATE SOURCE REGISTRY**
Confirm all sources used are registered in:
- `sources/drd_sources_[domain].md` (domain-specific)
- `sources/drd_sources_master.md` (central registry)

---

## FINAL QUALITY CHECK

```
[ ] Brief created and filed in briefs/
[ ] Destination department context file cross-linked
[ ] DRD_INDEX.md updated — stage moved to deployed
[ ] Domain counts updated
[ ] Source registry current
[ ] All pipeline files named correctly and in correct directories
```

---

## COMPLETE

> All six stages complete. All files filed. Index updated. Research is now part of the OS.

**DONE → return to the destination department to begin applying the findings.**
