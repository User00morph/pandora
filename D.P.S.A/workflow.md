# WF_DPSA_INNER_WORK
**D.P.S.A — Department of Personal Spiritual Autonomy & Personal Agency**
**Workflow: From Trigger to Sovereign Integration**

---

## WHAT THIS WORKFLOW IS

The process for doing structured inner work — whether it is processing a trigger, conducting shadow work, refining identity architecture, or deepening the observer's capacity. This is the most personal workflow in the OS. It applies rigorous structure to the most subjective domain.

---

## TRIGGER

- An emotional reaction that does not match the situation (signal of shadow material)
- A recurring pattern in behavior, relationships, or internal state
- A deliberate identity refinement session (scheduled or intuitive)
- A major life event that needs conscious processing
- A natal chart transit or cycle that calls for inner attention
- A question about perception, belief, or self-knowledge

---

## THE WORKFLOW — STEP BY STEP

### STAGE 1 — NOTICE (Nigredo)
**See what is actually happening inside.**

```
ACTION:
Document the trigger or inner event honestly:
- What happened externally? (if anything)
- What happened internally? (emotion, thought pattern, physical sensation)
- What is the automatic response you noticed?
- What is beneath the automatic response?
  (Often the first answer is a surface layer —
   ask "what is beneath that?" at least once)

Do not fix yet. Just see.

OUTPUT: dpsa_notice_[topic]_YYYY-MM-DD.md
```

---

### STAGE 2 — INVESTIGATE (Albedo)
**Trace the pattern to its source.**

```
ACTION:
1. PATTERN RECOGNITION: Is this a recurring pattern?
   When have you felt/reacted this way before?
   What situations trigger this specific response?

2. BELIEF AUDIT: What belief is driving this pattern?
   State the belief explicitly.
   Example: "If I show vulnerability, I will be exploited"
   Is this belief currently true, or is it inherited/outdated?

3. ORIGIN TRACE: Where did this pattern or belief form?
   Childhood? A specific event? Cultural programming?
   This is not about blame — it is about understanding
   the architecture that is currently running.

4. FUNCTION ANALYSIS: What purpose does this pattern serve?
   Every pattern — even destructive ones — was originally
   protective or adaptive. What was it protecting against?

SKILLS TO LOAD: skill_framework-synthesis.md (for building understanding)
OUTPUT: dpsa_investigation_[topic].md
```

---

### STAGE 3 — REDESIGN (Citrinitas)
**Build the replacement architecture.**

```
ACTION:
The old pattern was running on an old belief for an old reason.
Design the replacement:

1. NEW BELIEF: What is the upgraded belief that replaces the old one?
   It must be genuinely true — not an affirmation that feels hollow.

2. NEW RESPONSE: When the trigger occurs, what is the
   sovereign response instead of the automatic one?

3. PRACTICE: What practice reinforces the new pattern?
   (This may route to D.O.M for ritual design)

4. ANCHOR: What physical or environmental cue helps
   you remember the new pattern when the trigger hits?

OUTPUT: dpsa_redesign_[topic]_v1.md
```

---

### STAGE 4 — INTEGRATE (Rubedo)
**Live the new pattern and observe.**

```
ACTION:
- Implement the redesigned response in daily life
- Log observations: is the new pattern holding?
  When does it slip? What conditions make it easier/harder?
- Refine as needed — integration is iterative, not instant
- After sufficient testing (weeks, not days),
  assess whether the pattern has genuinely shifted

OUTPUT: dpsa_integration_[topic]_log.md
```

---

### STAGE 5 — ARCHIVE
**Document the completed inner work.**

```
ACTION:
- File the complete cycle (notice → investigate → redesign → integrate)
- Note what was learned that applies beyond this specific pattern
- Cross-link to D.O.M if a practice was designed
- Cross-link to D.B.S if body-level changes were involved

OUTPUT: dpsa_archive_[topic]_complete.md
```
