# Review Gate Checklist — Universal
**Human edit surface between pipeline stages**

---

## BEFORE ADVANCING ANY STAGE, CONFIRM:

### Output Quality
- [ ] The stage output exists and is complete
- [ ] The output addresses all items listed in the stage contract's PROCESS section
- [ ] The output is filed in the stage's `output/` folder

### Trace Integrity
- [ ] The output is consistent with inputs from the previous stage
- [ ] No claims or decisions were made that contradict earlier stage findings
- [ ] If the output changes direction from earlier stages, the reason is documented

### Reference Compliance
- [ ] All Layer 3 references listed in the stage contract were consulted
- [ ] Relevant skills were loaded and applied
- [ ] Source tiers are stated where evidence claims are made

### Readiness
- [ ] The output is ready to serve as input to the next stage
- [ ] Any ambiguity that would block the next stage has been resolved
- [ ] If this stage's output routes to another department, the handoff template is complete

---

*Review gates are where the human maintains sovereignty over the pipeline.*
