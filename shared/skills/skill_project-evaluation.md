# SKILL — PROJECT EVALUATION
**Loadable by: D.S.C, D.S.E**

---

## WHAT THIS SKILL IS

The alchemical intake process for evaluating whether a raw idea, opportunity, or impulse deserves to become a sanctioned project inside the OS. This skill protects the OS from three failure modes: chasing every idea without finishing anything, building the wrong thing with full effort, and starting without a clear enough picture to know when done.

Load this skill whenever a new idea arrives and needs honest evaluation before resources are committed.

---

## THE PROCESS

```
1. PLAIN STATEMENT
   Describe the idea in one sentence.
   Not the vision — the actual thing.
   "A YouTube channel about sovereign living"
   not "a media empire that changes consciousness."
   If this cannot be written — the idea is not ready.

2. ORIGIN TRACE
   Where did this come from?
   - A problem you experienced?
   - Something you consumed that sparked it?
   - A gap you noticed in the market?
   - An instruction from inner intelligence?
   - A department request?
   Trace honestly — origin reveals motivation.
   An idea that cannot explain where it came from
   cannot explain where it is going.

3. ALIGNMENT TEST
   Does this serve the Pandora Empire mission?
   Does it connect to at least one Triple Helix pillar
   (STIS / RWIS / PNC-AI) or the long-term sovereign estate vision?
   If it connects to nothing in the OS — why is it here?
   Score: ALIGNED / TANGENTIAL / DISCONNECTED

4. SOVEREIGNTY CHECK
   Does this expand or contract sovereignty?
   EXPANDS: generates income, builds skill, produces assets,
            increases freedom, serves others genuinely
   CONTRACTS: creates dependency, dilutes focus,
              serves someone else's agenda at sovereign cost
   Be honest. Some opportunities look like expansion
   and are actually contraction in disguise.
   Score: EXPANDS / NEUTRAL / CONTRACTS

5. RESOURCE REALITY
   What does this actually cost?
   - Time: hours per week, for how long?
   - Money: startup costs, ongoing costs?
   - Attention: what gets displaced if this enters the OS?
   - Energy: is the current bandwidth available?
   This test is non-negotiable. Must be answered clearly.

6. UNIQUENESS QUESTION
   What does Morph specifically bring to this
   that someone else cannot replicate easily?
   If the answer is nothing — reconsider.
   If the answer is clear — that is the foundation to build on.

7. TIMING QUESTION
   Why now? Is now actually the right time?
   The 2027 threshold is the strategic frame —
   does this idea serve the pre-threshold build
   or does it distract from it?
   Score: URGENT / TIMELY / CAN WAIT / WRONG TIMING

8. VERDICT
   Based on all seven steps:
   PROCEED → move to blueprint design
   HOLD → the idea has merit but the timing or resources are wrong
   DISSOLVE → the idea does not survive honest evaluation
   Document the reasoning for whichever verdict is given.
```

---

## QUALITY CHECK

Before a project evaluation is considered complete:

```
[ ] Plain statement fits in one sentence
[ ] Origin is traced honestly, not romanticized
[ ] Alignment is scored, not assumed
[ ] Sovereignty check confronted the uncomfortable possibility
[ ] Resource reality answered with specific numbers, not "manageable"
[ ] Uniqueness question has a concrete answer
[ ] Timing is assessed against the 2027 threshold
[ ] Verdict is given with documented reasoning
```

## OUTPUT

```
File: dsc_eval_[project-name]_v1.md
Contains: All 8 steps with honest answers
Verdict: PROCEED / HOLD / DISSOLVE with reasoning
```
