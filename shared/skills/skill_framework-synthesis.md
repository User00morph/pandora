# SKILL — FRAMEWORK SYNTHESIS
**Loadable by: D.R.D, D.S.C, D.S.S, D.I.I, D.P.S.A**

---

## WHAT THIS SKILL IS

The ability to take multiple research findings, decoded concepts, or raw observations and synthesize them into a coherent, usable framework. Loaded during the reconstruction phase of research or when building new conceptual architectures.

---

## THE PROCESS

```
1. GATHER THE RAW MATERIAL
   Collect all findings, decoded elements, and observations
   that relate to the framework being built.
   Do not filter yet — lay everything out.

2. IDENTIFY THE ORGANIZING PRINCIPLE
   What is the single principle that ties these elements together?
   The framework is built around this — not around the data.
   The data serves the principle; the principle explains the data.

3. MAP THE COMPONENTS
   What are the distinct parts of this framework?
   Name each component clearly.
   Each component should be describable in one sentence.

4. DEFINE THE RELATIONSHIPS
   How do the components interact?
   Which feeds which? Which depends on which?
   Draw the arrows — input/output between components.

5. TEST FOR COMPLETENESS
   Walk a real scenario through the framework.
   Does it handle the scenario from input to output?
   Where does it break? That gap is a missing component.

6. TEST FOR EXCESS
   Is any component doing nothing unique?
   If removing a component does not break the framework,
   it does not belong in the framework.

7. NAME IT
   A framework without a name is hard to reference and deploy.
   The name should describe what it does, not what it is about.
```

---

## QUALITY CHECK

```
[ ] Organizing principle is stated in one sentence
[ ] Each component is named and described
[ ] Relationships between components are mapped
[ ] At least one real scenario tested through the framework
[ ] No orphan components (every part connects to at least one other)
[ ] Framework has a clear, descriptive name
```
