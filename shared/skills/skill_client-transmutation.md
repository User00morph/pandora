# SKILL — CLIENT TRANSMUTATION
**Loadable by: D.S.E**

---

## WHAT THIS SKILL IS

The structured process for taking a client's raw problem and transmuting it into a functional sovereign architecture they can operate within. This is the core service delivery skill of D.S.E.

---

## THE PROCESS

```
1. INTAKE — RECEIVE THE RAW MATERIAL
   What is the client's problem, stated in their own words?
   What context have they provided?
   What data is available?
   Do not reframe yet — receive it exactly as presented.

2. DIAGNOSE — SEE THE ACTUAL STRUCTURE
   Beneath the client's description, what is actually happening?
   What system is producing the problem?
   What inputs are creating the outputs they do not want?
   Identify the structural cause, not just the symptom.

3. MAP — IDENTIFY WHAT NEEDS TO CHANGE
   Which components of their current system need to be:
   - Removed (producing distortion)
   - Added (missing entirely)
   - Redesigned (present but dysfunctional)
   - Reordered (correct parts, wrong sequence)

4. ARCHITECT — BUILD THE SOVEREIGN SOLUTION
   Design the replacement architecture:
   - New inputs
   - New processes
   - New outputs
   - Feedback loop for the client to self-correct going forward
   The solution must be operable by the client, not dependent on Morph.

5. DELIVER — PRESENT THE ARCHITECTURE
   Deliver the solution in a format the client can use:
   - Clear language, no unnecessary jargon
   - Step-by-step where needed
   - Visual mapping where it helps
   The deliverable is a functional structure, not advice.

6. HANDOFF — ENSURE INDEPENDENCE
   The client should be able to operate the solution independently.
   If ongoing support is needed — that is a separate engagement.
   The goal is sovereign function, not dependency.
```

---

## QUALITY CHECK

```
[ ] Client's original problem is documented in their own words
[ ] Structural diagnosis goes beyond symptoms
[ ] Solution is a system with inputs, processes, outputs, and feedback
[ ] Client can operate it independently
[ ] Deliverable format matches client's capacity to use it
```
