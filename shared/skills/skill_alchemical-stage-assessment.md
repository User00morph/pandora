# SKILL — ALCHEMICAL STAGE ASSESSMENT
**Loadable by: D.S.C, all departments**

---

## WHAT THIS SKILL IS

The ability to accurately determine which alchemical stage a system, project, or process is currently in — and whether it is genuinely ready to advance. Every department in the OS uses the four-stage alchemical lifecycle (Nigredo → Albedo → Citrinitas → Rubedo), but the most common failure is advancing a stage prematurely because momentum feels like readiness.

Load this skill when evaluating whether something is ready to move forward, when a project feels stuck and the stage is unclear, or during any audit that needs to confirm lifecycle position.

---

## THE PROCESS

```
1. CURRENT STATE CAPTURE
   What does this system/project look like right now?
   Describe the actual state — not the intended state,
   not the hoped-for state, not the last-documented state.
   What exists right now, today?
   
   Key questions:
   - What outputs has it produced so far?
   - What decisions have been made?
   - What decisions remain unmade?
   - What is unresolved or unclear?

2. STAGE IDENTIFICATION
   Match the current state against each stage's criteria:
   
   NIGREDO — Raw Material
   ☐ The idea/problem exists in unprocessed form
   ☐ No evaluation has been applied yet
   ☐ Raw material has been received but not examined
   ☐ Obstacles are unnamed, potential is undefined
   → If most boxes checked: this is Nigredo.
   
   ALBEDO — Purification
   ☐ The raw material has been documented
   ☐ Evaluation or testing is underway
   ☐ Some elements have been identified as serving / not serving
   ☐ Distortions are being identified and removed
   ☐ The shape is becoming clearer but design is not complete
   → If most boxes checked: this is Albedo.
   
   CITRINITAS — Illumination
   ☐ Purification is complete — what serves is separated from what doesn't
   ☐ The architecture or design is visible and documented
   ☐ Components are mapped, logic is structured
   ☐ A blueprint or plan exists that could be executed
   ☐ Resource requirements are known
   → If most boxes checked: this is Citrinitas.
   
   RUBEDO — Integration
   ☐ The design has been built and deployed
   ☐ The system is producing real outputs in the real world
   ☐ Feedback is being received from actual operation
   ☐ Maintenance and optimization are the current work
   → If most boxes checked: this is Rubedo.

3. ADVANCEMENT READINESS TEST
   Is this ready to move to the next stage?
   
   Nigredo → Albedo requires:
   ☐ Raw material is documented completely — nothing hidden
   ☐ A plain statement of what this is exists
   ☐ The idea can be evaluated — it is specific enough to test
   
   Albedo → Citrinitas requires:
   ☐ All purification tests have been run honestly
   ☐ What serves and what doesn't is clearly separated
   ☐ Resource reality is understood
   ☐ The idea has survived scrutiny — not just enthusiasm
   
   Citrinitas → Rubedo requires:
   ☐ A complete blueprint or design exists
   ☐ The minimum viable version is defined
   ☐ Resources are allocated and available
   ☐ First milestone is identified
   ☐ Kill conditions are set
   
   If any required box is unchecked — the stage is not complete.
   Do not advance. Complete the current stage first.

4. REGRESSION CHECK
   Has this system regressed from a later stage to an earlier one?
   Signs of regression:
   - A system in Rubedo that has stopped producing output → may have regressed to Citrinitas (design no longer fits reality)
   - A project in Citrinitas where the blueprint keeps changing → may need to return to Albedo (purification was incomplete)
   - Anything that feels stuck → check if the Nigredo was honest
   
   Regression is not failure. It is the system telling you
   which stage needs more work. Honor the signal.

5. STATUS UPDATE
   Based on the assessment:
   - Assign the correct stage marker
   - Document the reasoning
   - If advancement is blocked, name exactly what is missing
   - If regression occurred, name what triggered it
   - Update the relevant tracker (D.S.C project register or department context)
```

---

## STAGE MARKERS

```
🔲 NIGREDO    — raw, being received
⬜ ALBEDO     — being purified / tested
🔶 CITRINITAS — being designed / planned
🔴 RUBEDO     — active / in execution
✅ COMPLETE   — finished (output fully integrated)
❌ DISSOLVED  — stopped (with kill condition documented)
🔄 REGRESSED  — returned to earlier stage (with reason)
```

---

## QUALITY CHECK

Before an assessment is considered complete:

```
[ ] Current state described as-is, not as-intended
[ ] Stage identification uses the checkbox criteria, not gut feel
[ ] Advancement readiness checked against specific requirements
[ ] Regression possibility considered honestly
[ ] Status marker assigned and tracker updated
```
