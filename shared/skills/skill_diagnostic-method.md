# SKILL — DIAGNOSTIC METHOD
**Loadable by: D.S.M, D.S.C**

---

## WHAT THIS SKILL IS

A structured troubleshooting process for diagnosing mechanical, system, or process failures. While designed for vehicle diagnostics in D.S.M, the method applies to any system that has stopped producing the expected output. Loaded when something is broken and the cause is not immediately obvious.

---

## THE PROCESS

```
1. SYMPTOM DOCUMENTATION
   What is happening? Describe it precisely.
   - What does the operator observe? (sounds, smells, behaviors, outputs)
   - When did it start?
   - What changed before it started?
   - Is it constant or intermittent?
   - Are there conditions that make it better or worse?

2. SYSTEM IDENTIFICATION
   Which system(s) could produce this symptom?
   List all possibilities — do not jump to the most likely one yet.
   For vehicles: engine, electrical, fuel, cooling, drivetrain,
                 suspension, braking, HVAC, body
   For other systems: identify the equivalent subsystems.

3. ISOLATION
   Narrow down which system is actually responsible.
   - Test each candidate system independently where possible
   - Use elimination: if System A works normally in isolation,
     it is not the source
   - Work from simplest/cheapest to check toward most complex

4. ROOT CAUSE IDENTIFICATION
   Within the identified system, find the specific component
   or condition causing the failure.
   - What is the component supposed to do?
   - What is it actually doing?
   - What would cause that difference?

5. VERIFICATION
   Before replacing or repairing:
   - Can you reproduce the symptom reliably?
   - Does your diagnosis explain ALL the observed symptoms?
   - If it does not explain all symptoms, there may be
     multiple causes or the diagnosis is incomplete.

6. REPAIR / CORRECTION
   Fix the root cause, not the symptom.
   Document what was done and what was found.

7. CONFIRMATION
   After repair:
   - Does the symptom resolve?
   - Are there any new symptoms?
   - Run the system under the conditions that produced the original failure.
```

---

## RULES

```
- Never replace parts based on guessing — diagnose first
- The most common cause is not always the actual cause
- If the diagnosis does not explain all symptoms, keep looking
- Document everything — the next time this fails, the record saves hours
```
