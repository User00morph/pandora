# NIGREDO — Raw Material Extraction
**D.S.C | Stage 1 of 4 | The Prima Materia**

---

## PURPOSE
See the idea clearly before evaluating it. Strip the distortions of excitement and doubt. Receive what is actually there.

## INPUTS

| Source | Layer | Path |
|--------|-------|------|
| Raw idea / opportunity / impulse | Layer 4 (working) | Direct from Morph or department request |
| System Design skill | Layer 3 (reference) | `skills/skill_system-design.md` |
| Voice configuration | Layer 3 (reference) | `_config/voice.md` |
| Naming conventions | Layer 3 (reference) | `_config/naming-conventions.md` |

## PROCESS

1. **PLAIN STATEMENT** — Describe the idea in one sentence. Not the vision — the actual thing.
2. **ORIGIN TRACE** — Where did this come from? A problem? A spark? A gap? An inner instruction? Trace honestly — origin reveals motivation.
3. **PROBLEM/CREATION STATEMENT** — What problem does it solve? For whom? If no problem — what does it create or express?
4. **RAW OBSTACLES** — List every obstacle without problem-solving. Just name them.
5. **LANDSCAPE SCAN** — What already exists that is similar? Not a reason to stop — information about the terrain.

## OUTPUTS

| Output | Filename | Destination |
|--------|----------|-------------|
| Nigredo document — raw, unedited answers to all five items | `dsc_nigredo_[project-name]_raw.md` | `output/` |

## REVIEW GATE
**Before proceeding to Albedo, verify:**
- [ ] The plain statement fits in one sentence
- [ ] The origin trace is honest, not romanticized
- [ ] Obstacles are named without solutions attached
- [ ] If the plain statement cannot be written — the idea is not ready. Let it develop and return.

## CROSS-STAGE TRACE
- This is Stage 1 — no prior stage to verify against
- The plain statement from this stage becomes the anchor for all subsequent stages — if it changes later, that change must be documented
