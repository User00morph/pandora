# WF_DSC — STAGE 4: RUBEDO
**Integration — Bring the project into the OS officially.**

---

## WHAT THIS STAGE IS

A project that survived all three stages is sanctioned. This stage integrates it into the OS so it has a home, receives attention, and connects to the departments it needs. A project without a first milestone is an intention, not a project.

---

## ACTION — Integration Sequence

**1. CREATE THE PROJECT FILE**
File: `[dept-code]_project_[name]_v1.md`
Contents: The completed blueprint from Stage 3
Location: Primary department's folder

**2. UPDATE CONTEXT FILES**
- Add the project to the managing department's context file
- Add a reference in every department the project draws from

**3. SET THE FIRST MILESTONE**
From the Phase 1 architecture — what is the single first deliverable?
Name it. Assign it. Put a date on it.

**4. ASSIGN ALCHEMICAL STATUS**
Every active project carries a current stage marker:
```
🔲 NIGREDO     — raw, being evaluated
⬜ ALBEDO      — being purified / tested
🔶 CITRINITAS  — being designed / planned
🔴 RUBEDO      — active / in execution
✅ COMPLETE    — finished
❌ DISSOLVED   — stopped (kill condition documented)
```

**5. LOG IN DSC PROJECT REGISTER**
Add to `dsc_project_register.md`:
```
| PROJECT NAME | DEPT | STAGE | FIRST MILESTONE | DATE ENTERED | STATUS |
```

---

## OUTPUT

Active project file created. Context files updated. First milestone set. Project register updated.

File as: `dsc_project_[name]_v1.md`

---

## COMPLETE

> All four outputs exist → the project is officially inside the OS.
>
> NIGREDO doc + ALBEDO evaluation + CITRINITAS blueprint + RUBEDO integration = sanctioned project.

**DONE → close this workflow. Navigate to the project's home department to begin execution.**

---

## FAILURE MODES TO AVOID

```
SKIPPING NIGREDO       — purifying something never clearly defined
ROMANTIC ALBEDO        — passing tests because excited, not because they passed
NO MINIMUM VIABLE      — designing full vision, never starting
NO FIRST MILESTONE     — sanctioned project that never moves
NEVER DISSOLVING       — dead projects blocking new ones from entering
```
