# D.C.E Stage 4 — RUBEDO: Production and Transmission

## Purpose
The piece is created and released. Adapt from YouTube master to all platform formats. Capture field response as data for the next piece.

## Inputs
- Final script from `D.C.E/scripts/output/` (Stage 3 output)
- Load: `skill_content-production.md`
- Load: `D.C.E/_config/format-patterns.md` (all platform sections relevant to this piece)

## Process — Production
1. Confirm final script is complete and approved (mirror test, specificity test, voice test passed)
2. Identify all platform formats this piece will reach: YouTube → [TikTok / Reels / Threads / Substack / Podcast]
3. Produce YouTube master first. All others derive from this.

## Platform Adaptation Protocol
```
YouTube master → complete script + production notes
TikTok / Reels → extract highest-impact 60-90 second moment. New hook if needed.
Threads / X → distill to one tweet core claim + thread expansion. No summary.
Substack → expand to written form. Additional sources. No headers that recap.
Podcast → audio pace. Recap between sections. Shorter sentences on complex beats.
```

4. File each adaptation as: `dce_distribution_[title]_[platform].md`

## Process — Transmission
1. YouTube uploads first.
2. Other platforms receive from YouTube timeline (not simultaneously unless time-sensitive).
3. Monitor field response as information — not validation seeking.

## Field Response Capture
After first 48-72 hours of a piece being live:
```
PIECE: [title]
WHAT LANDED: [which angle/section got the most response]
WHAT MISSED: [what fell flat or got no engagement]
AUDIENCE INSIGHT: [what this reveals about what they need]
NEXT PIECE INFORMED: [how this feeds the next idea]
```

File to: `D.C.E/ideas/dce_field-response_[title]_[date].md`

## Done looks like
All platform adaptations filed. Piece transmitted on YouTube. Field response capture scheduled.

## Output goes to
`D.C.E/ideas/` (field response) → feeds back to Stage 1 (Nigredo)
