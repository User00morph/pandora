# D.C.E Stage 3 — CITRINITAS: Structure and Script

## Purpose
Produce the script or draft. The idea has been examined and refined. Now it finds its final form.

## Inputs
- Concept document from `D.C.E/concepts/output/` (Stage 2 output)
- D.R.D reconstruction if research-backed (load specific file — L4)
- ALWAYS load: `D.C.E/_config/constraints.md`
- Load on first draft: `D.C.E/_config/voice-and-tone.md`
- Load when format confirmed: `D.C.E/_config/format-patterns.md`
- Load: `skill_copywriting.md`

## Context Layer Protocol (before writing)
```
REFERENCE (L3 — apply as constraints, do not transform):
  _config/constraints.md
  _config/voice-and-tone.md
  _config/format-patterns.md — [specific format section only]

SOURCE (L4 — transform this into the script):
  dce_concept_[title]_v1.md
  [D.R.D reconstruction file if applicable]
```

## Process
1. Confirm format from concept document. Load the correct format-patterns.md section.
2. Write the opening — no warm-up. Enter the content immediately.
3. Build through the outline seed. Each beat earns its length — no padding.
4. Apply voice-and-tone.md throughout: layers, ancient-modern bridges, framework structure.
5. Check against constraints.md before completing. Fix every violation.
6. Mirror test: does this reflect something the audience can claim as their own?
7. Specificity test: are all claims grounded in specific evidence?

## Draft versions
- `_draft`: working version
- `_v2`, `_v3`: revision passes
- `_final`: cleared all quality checks

## Done looks like
A script or draft that passes all three tests: mirror test, specificity test, voice test. No constraint violations present. Ready for production.

## Output goes to
`D.C.E/scripts/output/dce_script_[title]_final.md` → Stage 4 (Rubedo)
