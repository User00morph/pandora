# WF_DCE_CONTENT_CREATION
**D.C.E — Department of Sovereign Content Creation & Expression**
**Workflow: From Raw Idea to Published Transmission**

---

## WHAT THIS WORKFLOW IS

The complete pipeline for taking a creative impulse and moving it through development, production, and publication. Every piece of content that exits the OS passes through this workflow — whether it is a YouTube video, a written piece, a social post, a course module, or a podcast episode.

---

## TRIGGER

Any of the following activates this workflow:

- A raw idea, observation, or creative impulse that wants to become content
- A scheduled content piece that needs to be produced
- A request from another department for expression (D.R.D decoded research, D.S.E client-facing content, D.S.S scientific breakdown)
- External event or moment that demands timely response

---

## INPUT REQUIRED

```
IDEA: What is the raw creative impulse?
MEDIUM: Video / written / audio / social / course / visual
AUDIENCE: Who is this for?
SOURCE DEPARTMENT: Where does the substance come from?
                   (personal experience / D.R.D research / D.S.S science / etc.)
TIMELINE: Time-sensitive or open?
```

---

## THE WORKFLOW — STEP BY STEP

### STAGE 1 — CAPTURE (Nigredo)
**Receive the idea without editing it.**

```
ACTION:
Write the idea down exactly as it arrived.
Do not structure it. Do not polish it. Do not evaluate it.
Just capture the raw impulse — one paragraph, a few bullet points,
a voice note transcription, whatever form it takes.

OUTPUT: Raw idea capture — filed as dce_capture_[topic]_raw.md
```

---

### STAGE 2 — BRIEF (Albedo)
**Define what this piece actually is before producing it.**

```
ACTION — Answer each:

1. CORE MESSAGE: What is the single most important thing
   this piece communicates? (one sentence)

2. AUDIENCE: Who specifically is this for?
   What do they already know? What do they need?

3. MEDIUM DECISION: Which medium serves this message best?
   (Not which is easiest — which is best for this specific content)

4. INTENDED OUTCOME: After consuming this, what should
   the audience think, feel, know, or do differently?

5. AUTHENTICITY CHECK: Is this coming from a genuine place?
   Would Morph stand behind this in any room?
   If not — either find the genuine version or shelve it.

6. DEPENDENCIES: Does this need research from D.R.D?
   Data from D.S.S? A framework from D.S.C?
   Route those requests now before production begins.

SKILLS TO LOAD: skill_copywriting.md
OUTPUT: Content brief — dce_brief_[topic].md
```

**BRIEF GATE:** If the core message cannot be stated in one sentence, the piece is not ready for production.

---

### STAGE 3 — DRAFT / SCRIPT (Citrinitas)
**Produce the first complete version.**

```
ACTION:
Write the piece in full — first pass, no editing during the draft.

For video: Full script or structured outline with key points,
           hooks, transitions, and call-to-action
For written: Complete first draft — intro through conclusion
For social: Complete post(s) with copy
For audio: Episode outline with talking points and flow
For course: Module structure with lesson content

SKILLS TO LOAD: skill_copywriting.md, skill_content-production.md
OUTPUT: First draft — dce_draft_[topic]_v1.md
```

---

### STAGE 4 — REVIEW (Purification)
**Evaluate the draft against the brief.**

```
ACTION — The five review checks:

1. MESSAGE FIDELITY: Does the draft deliver the core message
   from the brief? Or has it drifted?

2. AUDIENCE FIT: Is the language, depth, and tone right
   for the intended audience?

3. CUT TEST: Is anything included that does not serve
   the core message? If so — cut it.

4. VOICE CHECK: Does this sound like Morph?
   Authentic, direct, grounded, no filler, no performance?

5. STRUCTURE CHECK: Can someone follow this from start to finish
   without getting lost?

OUTPUT: Reviewed draft — dce_draft_[topic]_v2.md (or v1 with edits marked)
```

---

### STAGE 5 — PRODUCTION (Rubedo)
**Produce the final version for publication.**

```
ACTION:
Produce the piece in its final publishable form.

Video: Record, edit, create thumbnail, write metadata/description
Written: Final edit, formatting, images if needed
Social: Final copy, visual assets, scheduling
Audio: Record, edit, write show notes
Course: Final module assembly, supporting materials

OUTPUT: Final piece — dce_[medium]_[title]_final.md
        + all supporting assets filed alongside
```

---

### STAGE 6 — PUBLISH AND FILE
**Release and archive.**

```
ACTION:
1. Publish to the appropriate platform(s)
2. File the final version and all drafts in D.C.E
3. Cross-link to any source department
   (if research came from D.R.D, link the content to the research file)
4. Log in content register if one is maintained

OUTPUT: Published piece + complete file trail
```

---

## FAILURE MODES TO AVOID

```
PUBLISHING FROM DRAFT
Skipping Stage 4 because the draft "feels done."
The review stage is where quality is made. Never skip it.

BRIEF DRIFT
Starting production without a clear brief, or drifting
from the brief during production without updating it.

CONTENT WITHOUT SOURCE
Producing content that makes claims without routing
through D.R.D for evidence. Expression is authentic —
claims need evidence.

PERFECTIONISM LOOP
Cycling through drafts endlessly without publishing.
Done and honest beats polished and unseen.
```
